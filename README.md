## Данные

Данные, которые я использую в этом проекте, взяты c [Kaggle](https://www.kaggle.com/datasets/thedevastator/netflix-imdb-scores). Это англоязычный датасет с краткими анонсами/сюжетами фильмов и сериалов. Оригинальный датасет лежит в `data/Netflix TV Shows and Movies.csv`.

Суть данного проекта –– реализация поиска по описанию фильмов/сериалов. По этой причине изначальный датасет был предобработан: все строки, содержащие `nan` в столбце `description`, были удалены. Дополнительно, я избавилась от не очень информативных для этого задания столбцов. Новый датасет лежит в `data/data.csv`.

Всего в коллекции хранится $5277$ документов.

Каждый текст коллекции был токенизирован, лемматизирован и отчищен от пунктуации при помощи `spacy`. Итоговый корпус вместе с метаданными размещен в `data/metadata.json`.

## Индексация

В данном проекте реализованы три вида индексации:

1) `BM-25`

2) `wor2vec` –– [реализация `gensim`](https://radimrehurek.com/gensim/models/word2vec.html)

3) `fasstext`  –– [реализация `gensim`](https://radimrehurek.com/gensim/models/fasttext.html#gensim.models.fasttext.FastText)


Для каждого вида индекса данные были векторизованы, а модели / векторизаторы / матрицы документов были сохранены в директорию `models`. В дальнейшем они использовались для векторизации запроса.

Вектор документа для индексов `wor2vec` и `fasstext` считался как усредненный вектор всех слов в документе.

Дополнительно перед обработкой и векторизацией запроса он проверяется на опечатки при помощи библиотеки [pyspellchecker](https://pyspellchecker.readthedocs.io/en/latest/quickstart.html).

## Структура репозитория

Репозиторий устроен следующим образом:

```
├── README.md
├── app.py
├── data
│   ├── Netflix TV Shows and Movies.csv
│   ├── data.csv
│   └── metadata.json
├── environment.yml
├── index.py
├── models
│   ├── bm25_matrix.npz
│   ├── bm25_vectorizer.pkl
│   ├── fasttext.model
│   ├── fasttext_matrix.npy
│   ├── word2vec.model
│   └── word2vec_matrix.npy
├── preprocess.py
├── static
│   └── styles.css
└── templates
    ├── base.html
    ├── main.html
    ├── results.html
    └── search.html
```

За веб-интерфейс отвечают директории `templates` и `static` и скрипт `app.py`, который рендерит html-страницы.

Поиск реализован в скрипте `index.py`. В нем используются данные из корпуса (хранятся в директории `data`), сохраненные модели / матрицы / векторизаторы (хранятся в директории `model`).

Вся предобработка данных (в том числе и построение индексов) проводилось при помощи скрипта `preprocess.py`.

В `environment.yml` хранится информация о необходимом для проекта окружении.

## Запуск кода

Перед использованием кода необходимо создать виртуальное окружение и установить все библиотеки из `environment.yml` при помощи следующих команд.

```
conda env create -f environment.yml
conda activate project
python -m spacy download en_core_web_sm
```

### Использование сайта

Для того чтобы запустить сайт и использовать визуальный интерфейс, реализованный при помощи `Flask` необходимо запустить `app.py` и следовать инструкциям в веб-интерфейсе. 

```
python app.py
```

### Использование терминала

Для того чтобы работать c CLI и получить результаты поиска необходимо подготовить три аргумента, которые будут переданы со следущими флагами:

- `--query` –– отвечает за строку запроса
- `--idx` –– отвечает за тип индекса; может быть только `bm25`, `word2vec` или `fasttext`
- `--top` –– отвечает за тип число документов в выдаче

Пример использования CLI:

```
python index.py --query "dream nightmare sandman" --idx "bm25" --top 10
```

Для вывода инструкции к флагам внутри терминала используйте:

```
python index.py --help
```

CLI был реализован при помощи [`argparse`](https://docs.python.org/3/howto/argparse.html).
